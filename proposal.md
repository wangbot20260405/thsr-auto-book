# Proposal: 自動化高鐵訂票系統（thsr-auto-book）

## 1. 需求提出者

**歐郎**（Discord: karta571110）

---

## 2. 專案背景與動機

- 目標：打造一個可在 Linux 環境運行的自動化高鐵訂票系統
- 使用場景：適用於特定路線的定期訂票需求，可監控座位並在有空位時自動搶票
- 觸發方式：手動執行，提供查詢條件後，系統自動監控直到訂票成功或被中斷
- 技術棧：Python + Puppeteer（瀏覽器自動化），Git 版控，推送至 GitHub

---

## 3. 需求範圍

### 3.1 核心功能

1. **班次查詢**：根據起訖站、日期、出發時間查詢可用班次與座位
2. **互動式選擇**：將有空位的班次以清單形式發送至 Discord，供使用者選擇
3. **自動監控**：若選擇的班次無位，每 30 秒自動重新查詢直到有位
4. **自動訂票**：選定班次後，自動填入乘客資料並完成訂票流程
5. **驗證碼處理**：在驗證碼頁面前暫停，等使用者在 Discord 輸入驗證碼後繼續
6. **結果通知**：訂票成功後發送取票代碼至 Discord；失敗時發送錯誤資訊
7. **中斷機制**：使用者可隨時中斷監控

### 3.2 初期不納入（MVP 後再說）

- 不處理線上付款（假設付款自行至 App/機台操作）
- 不支援退票/改票
- 不做 OCR 自動過驗證碼

---

## 4. 技術架構

| 項目 | 選擇 |
|------|------|
| 語言 | Python 3 |
| 瀏覽器自動化 | Puppeteer（Python via pyppeteer） + Xvfb + Headful Chrome（Linux） |
| Discord 通知 | Discord Webhook（传入消息） |
| 驗證碼處理 | 停在驗證碼頁面，使用者在 Discord 輸入驗證碼後繼續 |
| 版控 | Git |
| GitHub | 新建 `thsr-auto-book` repo，Personal Access Token 認証 |
| 執行環境 | 本機 Linux（家裡） |

---

## 5. 使用者輸入（CLI 互動式）

腳本執行時收集：
1. 起站（如：台北）
2. 訖站（如：左營）
3. 出發日期（YYYY/MM/DD）
4. 出發時間範圍（HH:MM，起搜範圍）
5. 車廂種類（標準車廂對號座 / 商務車廂）
6. 票種與張數（全票、孩童票、敬老票、愛心票、大學生票）
7. 乘客身分證字號（必填）
8. Discord Webhook URL（或使用預設值）

---

## 6. 輸出

- Discord 訊息：有空位的班次清單（含車次、時間、剩餘座位數）
- Discord 訊息：訂票結果（成功 → 取票代碼 / 失敗 → 錯誤原因）
- 終端機日誌：便於本地 debug
